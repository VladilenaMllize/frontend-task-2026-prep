# DevJobs – Frontend Starter Project

## Overview

**DevJobs** е уеб приложение за управление на обяви за работа.

Приложението позволява:
- преглед на списък с обяви
- разглеждане на детайлна информация
- създаване на нова обява

Проектът е подготвен като стартова основа и следва структура, близка до реална Frontend работна среда.

---

## Tech Stack

- **React** (Vite)
- **React Router**
- **CSS Modules**
- **Axios**
- **JSON Server** (локален mock backend)

---

## Prerequisites

Преди да стартирате проекта, трябва да имате инсталирани:
- **Node.js** (версия 16 или по-нова)
- **npm**

Проверка (в терминала):
```bash
node -v
npm -v
```

---

## Installation

В основната директория на проекта изпълнете:
```bash
npm install
```

---

## Running the Application

Проектът може да бъде стартиран с една команда:
```bash
npm run dev:all
```

Тази команда стартира едновременно:
- **Frontend:** [http://localhost:5173/](http://localhost:5173/)
- **Backend (JSON Server):** [http://localhost:4000/jobs](http://localhost:4000/jobs)

### Стартиране поотделно (при нужда)

**Backend:**
```bash
npm run server
```

**Frontend:**
```bash
npm run dev
```

## Project Structure

Основната логика се намира в папка `src/`.

```text
src/
  ├── pages/        # Основни страници (routes)
  ├── components/   # Функционални компоненти
  ├── ui/           # Преизползваеми UI елементи
  ├── api/          # Комуникация с backend
  └── styles/       # Глобални стилове (tokens, base)
```

Локалната база данни се намира в:
```text
server/db.json
```

---

## Architecture Overview

Проектът следва компонентно-базирана архитектура:

- **Страниците** управляват зареждането на данни.
- **API слоят** съдържа всички HTTP заявки.
- **Компонентите** визуализират информацията.
- **UI компонентите** са преизползваеми и нямат бизнес логика.
- Страниците не комуникират директно с backend, а използват функциите от `api/`.

---

## Styling Approach

Проектът използва **CSS Modules**.

Всеки компонент има собствен `.module.css` файл, което осигурява:
- локалност на стиловете
- избягване на глобални конфликти
- по-добра поддръжка

Глобалните стилове са разделени на:
- `tokens.css` – централизирани design стойности (CSS променливи)
- `base.css` – базови стилове за приложението

---

## Development Guidelines

При добавяне на нова функционалност следвайте тези правила:

- Нови страници → `pages/`
- Нови функционални компоненти → `components/`
- Преизползваеми UI елементи → `ui/`
- API логика → `api/`

Стремете се да запазите структурата и архитектурния модел на проекта.

## Notes

- JSON Server използва `db.json` като локална база данни и симулира REST API.
- Уверете се първо, че JSON Server e стартиран, в случай че имате проблеми при изпълнението на заявките.
- Ако възникнат проблеми при стартиране:
  - Уверете се, че сте изпълнили `npm install`
  - Проверете дали портовете **5173** и **4000** не са заети
  - Проверете конзолата на браузъра за грешки
